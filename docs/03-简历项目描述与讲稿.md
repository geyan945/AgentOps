# AgentOps 简历项目描述与讲稿

## 简历项目描述

**AgentOps：基于 MCP 的企业级多工具智能体平台**

技术栈：Java 25、Spring Boot 3.5.11、Spring Security、Spring Data JPA、MySQL、Redis、RabbitMQ、Elasticsearch、Gemini API

项目描述：面向企业知识库与数据分析场景设计的多工具智能体平台，支持会话管理、Agent Run / Step Trace、工具调用编排、MCP 第一版协议接入与异步评测闭环。平台内置 `kb_search`、`doc_fetch`、`sql_query` 三类工具，复用 AISmartQA 的知识库与索引能力，并通过结构化规划与最终回答链路实现完整 Agent 执行流程。

项目亮点：
- 设计并实现 `session -> run -> step` 执行模型，沉淀 Agent Trace 数据，支持对单次执行全过程回放。
- 抽象统一 `ToolExecutor` 接口，完成本地工具与 MCP 远程工具双模式接入，降低后续新增工具和协议升级成本。
- 复用 AISmartQA 的知识库数据与 ES 索引，实现 `kb_search` / `doc_fetch` 工具，形成项目间能力复用。
- 基于 RabbitMQ 构建异步 Eval 流程，支持评测数据集、批量运行、结果统计和失败样本分析。
- 使用 Redis 实现运行限流、工具短期缓存和会话摘要缓存，增强平台稳定性与演示体验。

## 1 分钟介绍

我做了一个叫 AgentOps 的企业级多工具智能体平台。它不是单纯的 RAG，而是在知识库检索能力之上又加了一层 Agent 编排。系统支持用户登录、会话管理、Agent Run 和 Step Trace，同时内置 `kb_search`、`doc_fetch`、`sql_query` 三个工具。用户提问后，Planner 会先决定调用哪个工具，再把工具结果回填上下文生成最终回答。整个过程会完整落到 Run / Step，方便回放和评估。我还补了 MCP 第一版、异步 Eval、Redis 限流和缓存，使它更像一个企业级 AI 应用平台。

## 3 分钟介绍

AgentOps 的定位是企业级多工具智能体平台。和 AISmartQA 这种工程化 RAG 项目不同，AgentOps 不是只做问答，而是把检索和结构化查询都抽象成工具，由 Agent 根据用户问题动态选择。当前平台支持 JWT 登录、会话管理、Agent Run、Step Trace、MCP 第一版协议、异步 Eval，以及 `kb_search`、`doc_fetch`、`sql_query` 三类工具。

主链是：用户登录后创建会话，发起一次 Agent Run，系统先用 Planner 判断是否需要调用工具，然后执行本地工具或 MCP 远程工具，再把工具结果回填给 Answer 阶段生成最终回复。同时每一步都会在 `agent_run_step` 里记录输入、输出、耗时和错误信息。为了支撑离线质量验证，我还补了 Eval 数据集、RabbitMQ 异步评测、结果统计和失败样本分析。

项目里我特别强调三点。第一是工具抽象，所有工具统一走 `ToolExecutor` 接口，这样以后新增工具或者把服务拆成独立 MCP Server 都不需要重写主流程。第二是 Trace 与 Eval，可回放一次 Agent 执行的全部步骤，也能批量跑评测并沉淀质量数据。第三是平台治理，我加了 Redis 限流、工具缓存和会话摘要缓存，让它不只是“能调模型”，而是一个可治理的 Agent 平台。
