# AgentOps 10 分钟稳定讲稿与追问树

## 一. 10 分钟稳定讲稿

### 1. 项目定位
AgentOps 是一个企业级多工具智能体平台，不是普通问答系统。它的目标是把大模型、知识库检索和结构化数据查询放进统一的 Agent 执行框架里，让系统具备会话、Run/Step Trace、工具编排、MCP 扩展和异步评测能力。

### 2. 为什么要做这个项目
因为我已经有 AISmartQA，如果再做一个普通知识库问答项目，差异化不够。所以 AgentOps 选择站在 AISmartQA 的检索能力之上，重点补多工具调用、Agent 编排、可观测性和平台化扩展。

### 3. 当前架构
当前落的是单体平台版：
- Auth：JWT 登录与鉴权
- Session：会话、消息、摘要缓存
- Agent：Run / Step / Planner / Executor / Answer
- Tool：`kb_search`、`doc_fetch`、`sql_query`
- MCP：`initialize`、`tools/list`、`tools/call`
- Eval：数据集、异步 run、Dashboard、失败样本分析

### 4. 主执行链
完整链路是：
`用户提问 -> Planner -> Tool Call -> Final Answer -> Trace`

Run 表示一次完整执行，Step 记录 PLAN、TOOL_CALL、FINAL_ANSWER 三类步骤。这样系统不只是给结果，而是能回放中间过程。

### 5. 三个工具怎么设计
- `kb_search`：知识库检索，复用 AISmartQA 的数据和 ES 索引
- `doc_fetch`：读取指定文档内容
- `sql_query`：执行只读白名单模板查询，用于统计问题

我把它们统一抽象成 `ToolExecutor`，后续可以切成本地实现，也可以切到 MCP 远程工具。

### 6. MCP 第一版怎么落地
我没有停留在文档层，而是真的做了最小 MCP 能力：
- `initialize`
- `tools/list`
- `tools/call`

同时主系统里有 `McpHttpClient` 和 `kb_search_remote / sql_query_remote` 两个远程工具执行器，说明这条协议链已经可跑。

### 7. Eval 为什么重要
平台不能只会执行，还要会评测。我做了 Eval 数据集、Eval Run、Eval Result，并通过 RabbitMQ 异步跑 case，最后再在 Dashboard 里看通过率、平均分、平均耗时和失败样本。这让 AgentOps 更像平台，而不是单次聊天应用。

### 8. Redis 做了什么
Redis 在这里不是只当缓存。当前做了三类事情：
- 运行限流
- 工具结果短期缓存
- 会话摘要缓存

这样平台既有保护能力，也有一定的上下文治理能力。

### 9. 项目价值
这个项目最重要的不是“调了模型”，而是把模型和工具放到了一个可治理、可扩展、可评测的平台里。后续无论继续拆 MCP Server，还是升级更强 Planner/Answer，都有很清晰的演进路径。

## 二. 追问树

### 1. 为什么先做本地工具，再做 MCP？
答：因为本地工具更适合先把链路做完整。MCP 是协议层增强，应该建立在稳定的工具抽象和执行链之上，而不是一开始就让系统复杂度爆炸。

### 2. 为什么工具必须抽象成接口？
答：因为 Agent 主流程不应该依赖工具的具体实现方式。统一接口后，工具就可以自由切换为本地实现、HTTP 服务，或者 MCP Server，而主流程几乎不动。

### 3. MCP 第一版和真正拆服务版有什么区别？
答：当前是同进程演示版，重点是把协议能力和远程工具调用链落出来；真正拆服务版会把 `kb_search`、`sql_query` 单独拆成独立 MCP Server。也就是说，当前已经完成了协议与调用模式，后面再做的是部署形态拆分。

### 4. 为什么 Eval 要走 RabbitMQ 异步？
答：因为 Eval 是天然批处理场景，一次 Run 可能包含很多 case。同步跑会直接拖慢接口延迟，也会和主聊天链路争资源。异步后，用户发起请求能立即返回，后台慢慢消费 case 并累计统计结果，更像真实平台设计。

### 5. 会话摘要缓存的作用是什么？
答：它是平台向长会话治理迈的一步。当前先缓存最近 N 条消息的摘要，后续无论做上下文压缩、Prompt 注入还是多轮记忆优化，都可以在这层能力上继续演进。

### 6. AgentOps 和 AISmartQA 的关系怎么讲？
答：AISmartQA 提供知识库与检索底座，AgentOps 在其上增加会话、Run/Step、工具编排、MCP 和 Eval。可以理解成 AISmartQA 是能力底座，AgentOps 是上层智能体平台。

### 7. 如果面试官问这个项目下一步怎么升级？
答：先把 MCP 从同进程演示版拆成独立 server，再把 Planner 和 Answer 切到更强的 Gemini/Function Calling，最后把 Eval 升级成更强的质量评估体系。这几步都是平台增强，而不是推翻重写。
