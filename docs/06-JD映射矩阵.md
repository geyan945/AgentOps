# AgentOps JD 映射矩阵

## 一句话定位

AgentOps 当前对外统一口径是：**面向企业知识与数据 Copilot 的可治理 Agent 系统**。  
核心不是“会调模型”，而是 `Java control plane + Python LangGraph runtime + React workspace` 下的 `checkpoint / replay / approval / runtime Eval / tool governance / event stream`。

## 关键词映射

| JD 关键词 | AgentOps 对应实现 |
| --- | --- |
| `LangGraph` | Python runtime 主链是 `intake_guardrail -> load_memory -> supervisor_plan -> knowledge_researcher / data_analyst -> evidence_reviewer -> human_approval -> finalize` |
| `LangChain` | runtime 通过 `StructuredTool + Gemini LLM service` 封装模型节点和工具调用 |
| `Python / FastAPI` | `agentops-runtime-py` 对外暴露 `/runs / resume / replay / health` |
| `RAG / Tool Orchestration` | `kb_search / doc_fetch / sql_query` 由 supervisor 路由、reviewer 审核 |
| `Checkpoint / Resume` | `agent_runtime_checkpoint + checkpointVersion + resumeToken` |
| `Crash Replay / Recovery` | 非审批态支持从最新 checkpoint 做 replay / continue |
| `Human-in-the-loop` | `human_approval + approval inbox + AUTO_APPROVE / AUTO_REJECT / MANUAL` |
| `Skills` | runtime 记录 `knowledge_skill / data_skill / review_skill / finalize_skill` |
| `Multi-Agent / Team Graph` | `orchestrationMode = SINGLE_GRAPH / TEAM_GRAPH`，TEAM_GRAPH 是受控 multi-role 编排 |
| `Eval / Benchmark` | Eval 直接跑真实 runtime 图，保留 `route / grounding / citation / approval / nodePath / skills / latency / retry` |
| `Observability / Trace` | `run / step / graph / event stream / failure samples / dashboard` |
| `MCP` | Java control plane 暴露 MCP tool plane，Python runtime 通过工具调用接入 |
| `Tool Governance` | 工具元数据包含 `riskLevel / approvalPolicy / timeoutBudget / retryPolicy / auditRequired` |

## 对应用岗的讲法

- 强调 `RAG + tools + review + approval + eval` 这条业务主链。
- 强调自己能把需求翻译成 `single-graph` 或 `team-graph` 工作流。
- 强调 SSE 工作台、审批面板、Eval dashboard 这些“能直接演示”的证据。

## 对平台岗的讲法

- 强调 Java 是系统记录源，Python 只负责 runtime，不直接读数据库。
- 强调 `checkpoint / replay / approval / event stream / tool governance / runtime-based eval`。
- 明确当前是“受控多角色 team-graph”，不是完全开放式通用 multi-agent 平台。

## 诚实边界

- 不是官方 LangGraph durable checkpointer 接法，而是 Java-backed checkpoint store。
- 前端虽然已经支持 SSE，但轮询仍然保留为兼容兜底。
- 权限、租户、规模化运维还在增强，不把当前版本说成完全生产级 SaaS 平台。
